## Creating a new migration

Always create new migrations from the current head (`rename_title_to_rq`
until a newer revision lands):

```bash
uv run alembic revision --autogenerate -m "Description of changes"
```

Review the generated file under `versions/`, then apply:

```bash
uv run alembic upgrade head
```

## Fresh database

On an empty database:

```bash
uv run alembic upgrade head
```

`POSTGRES_PASSWORD` must be set when applying the initial migration. The
`init_full_schema` upgrade creates the `openscientist_admin` role and uses
that environment variable for the role password.

## Rollback (use with care)

```bash
uv run alembic downgrade -1              # Go back one revision
uv run alembic downgrade <revision_id>  # Go back to a specific revision
```

Downgrading through `init_full_schema` drops the full schema.
